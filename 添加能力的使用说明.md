## 架构总览

### 分层结构

```
main.py
  │
  ├── HelperEngine          AI助手引擎
  │     └── Agent ──────── ASSISTANT_SKILLS
  │
  └── InterviewEngine       面试引擎
        └── Agent ──────── INTERVIEW_SKILLS
```

```
UI层                        引擎层                    工具层
─────────────────────       ─────────────────────     ─────────────────────────────
AgentPanel                → HelperEngine             → ASSISTANT_SKILLS
                               Agent(assistant)          COMMON_GROUP
                                                         QUIZ_GROUP
                                                         RAG_GROUP        → search_knowledge_base → KnowledgeCore(tech_kb)
                                                         SEARCH_GROUP     → web_search, search_wikipedia
                                                         HISTORY_GROUP    → get_student_interview_history
                                                                          → get_student_id_by_name

InterviewPanel            → InterviewEngine          → INTERVIEW_SKILLS
                               Agent(interview)          COMMON_GROUP
                                                         DS_COURSE_GROUP  → search_ds_course → KnowledgeCore(ds_course_kb)
```

### 各层职责

**`main.py`** — 组装层，实例化 KnowledgeCore、引擎、UI，只做连线，无业务逻辑

**引擎层**（`helper_engine.py` / `interview_engine.py`）— Agent 的使用方，持有 Agent 实例，负责业务状态（面试 session、对话历史）和 system prompt 管理

**`agent_core.py`** — 纯 LLM + tool_calling 框架，不含业务逻辑，接受任意工具集，对外暴露 `stream()` / `chat()` / `clear_conversation()`

**工具层** (`service/tools/`)：
- `permissions.py` — ToolGroup 路由组定义 + SkillSet 组合，**只声明"谁有哪些工具"，不含实现**
- `registry.py` — 工具工厂的组装中心，按 SkillSet 筛选并返回工具列表
- `db_tools.py` — 所有数据库类工具的工厂函数
- `search_tools.py` — 联网搜索类工具的工厂函数
- `knowledge/` — 知识库能力子模块
  - `KnowledgeCore.py` — RAG 检索 API 封装，支持官方 SDK / HTTP 双模式
  - `create_knowledge_search_tool.py` — `search_knowledge_base`（HelperEngine 用）
  - `create_ds_course_tool.py` — `search_ds_course`（InterviewEngine 用）

## 添加新能力的步骤

### 情况一：新增一个普通工具（如新的 DB 查询、新的搜索能力）

**第一步** — 在对应文件写工厂函数

```python
# db_tools.py 或 search_tools.py
def create_xxx_tool(db):
    @tool(args_schema=XxxInput)
    def your_tool_name(...) -> str:
        """工具描述，模型靠这段话决定是否调用"""
        ...
    return your_tool_name
```

**第二步** — 在 `permissions.py` 加常量和分组

```python
# 加常量
TOOL_XXX = "your_tool_name"

# 加进合适的 ToolGroup（或新建一个）
HISTORY_GROUP = ToolGroup(
    name="history",
    tools=frozenset({TOOL_HISTORY, TOOL_STUDENT_LOOKUP, TOOL_XXX}),  # ← 加这里
)
```

**第三步** — 在 `registry.py` 注册工厂

```python
# build_tools() 里对应分类下加一行
("your_tool_name", create_xxx_tool, db, "db"),
```

SkillSet 自动更新，无需改引擎或 UI。

### 情况二：新增一个知识库（新的 RAG 检索能力）

**第一步** — 在 `knowledge/` 下新建工具文件

```python
# knowledge/create_xxx_kb_tool.py
def create_xxx_kb_tool(kb: KnowledgeCore):
    @tool(args_schema=XxxKbInput)
    def search_xxx(...) -> str:
        """描述这个知识库的内容和适用场景"""
        ...
    return search_xxx
```

**第二步** — 在 `knowledge/__init__.py` 导出

```python
from .create_xxx_kb_tool import create_xxx_kb_tool
```

**第三步** — `permissions.py` 加常量和 Group

```python
TOOL_RAG_XXX = "search_xxx"

XXX_KB_GROUP = ToolGroup(
    name="xxx_kb",
    tools=frozenset({TOOL_RAG_XXX}),
)
```

**第四步** — `registry.py` 的 `build_tools()` 加参数和工厂

```python
# 函数签名加参数
def build_tools(db=None, tech_kb=None, ds_course_kb=None, xxx_kb=None):

# 知识库工厂列表加一行
("search_xxx", create_xxx_kb_tool, xxx_kb, "xxx_kb"),
```

**第五步** — 把新 Group 加进目标 SkillSet，并在引擎构造时传入 `xxx_kb`

```python
# permissions.py
ASSISTANT_SKILLS = SkillSet.from_groups(
    "assistant", "...",
    COMMON_GROUP, QUIZ_GROUP, RAG_GROUP, XXX_KB_GROUP,  # ← 加这里
    SEARCH_GROUP, HISTORY_GROUP,
)
```

```python
# helper_engine.py 或 interview_engine.py
tools = get_tools_for(db=db, tech_kb=tech_kb, xxx_kb=xxx_kb, skill_set=ASSISTANT_SKILLS)
```

### 情况三：新增一个引擎（新的业务场景，如练习引擎）

1. 新建 `practice_engine.py`，持有 `Agent`，定义新的 SkillSet 或复用现有的
2. 在 `permissions.py` 按需新增 SkillSet 组合
3. 在 `UI/` 新建对应 Panel，接入引擎接口
4. 在 `main.py` 实例化引擎和 Panel，挂载到 TabWidget

---

### 一句话总结

> 加工具 → 改 `permissions`（声明归属）+ 改 `registry`（注册工厂）
> 加知识库 → 多加一个 `knowledge/` 文件 + 上面两步 + 引擎构造传参
> 加引擎 → 新建引擎文件 + 新建 Panel + main.py 连线
